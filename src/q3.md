**(c)** The SCTP course assumes a POSIX shell (bash or zsh). In your course material, you may encounter commands like these:

```bash
touch notes.txt
export API_KEY=abc123
chmod +x setup.sh
source venv/bin/activate
```

Give **one concrete reason** why running commands like these on Windows PowerShell can fail, even though some commands look similar to PowerShell at first glance. A general-knowledge answer is fine; you do not need to have used PowerShell yourself.

**Answer:**
_(PowerShell utilizes entirely different command parsing, built-in utilities, and syntax structures compared to POSIX shells like bash or zsh. For example, commands like export and chmod do not exist in PowerShell, which handles environment variables via a unique drive notation (e.g., $env:API_KEY="abc123") and manages file permissions through Windows Access Control Lists (ACLs) rather than POSIX permission bits)_
