**(a)** What is a **kernel**, in plain language? (One or two sentences. No textbook definition copied in; explain it as you understand it.)

**Answer:**
_(write your answer here)_The kernel is the core engine of the operating system that acts as a bridge between your software and your actual hardware. It runs behind the scenes to manage system resources, like deciding which program gets to use the CPU, how memory is allocated, and how files are written to the disk.

---

**(b)** A classmate says:

> "I installed WSL, so now I have two completely separate operating systems, Windows and Linux, running side by side, each totally independent."

Is this accurate? Briefly explain why or why not.

**Answer:**
_(write your answer here)_This is not entirely accurate because they are not totally independent. WSL (Windows Subsystem for Linux) runs a lightweight Linux kernel directly inside Windows, meaning the Linux environment highly integrates with the Windows host, sharing the same underlying hardware resources, network, and even allowing you to access your Windows files directly from the Linux terminal.

---

**(c)** Explain the difference between a **terminal (emulator)** and a **shell**, and give one example of each.

**Answer:**
_(write your answer here)_A terminal emulator is the visual window or desktop application that captures your keyboard inputs and displays the text output on your screen (e.g., Terminal on macOS or Windows Terminal). The shell is the actual command-line program running inside that window that interprets your typed commands and executes them (e.g., Bash or Zsh).
